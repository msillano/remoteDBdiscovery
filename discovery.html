Discovery protocols for remotesDB

Here the workflow to process a new unknown remote control, using some CRUD pages. You can do all required operations starting from the discovery page.

You must fallow the steps:
  1. You add a new record for the remote control filling all fields with required data. (You can also click [add new] idremote.)

  2. Now you must add a device, real or virtual. You do not need any real device to test a new remote control, but a device is required by remotesDB structure. (You can also click [add new] iddevice.)

  3. Add also an 'unknownXX' protocol (or any name you like) for the new control and do not fill the IRP field. (You can also click [add new] idprotocol.)

  4. Associate the remote control, the new device and the unknown protocol adding a new record to irp_devrem. code is mandatory, use '0' as default , or a code starting with an alpha char (A..Z). Use 'A' as default mode1, or set the modes that the device can accept.

  5. Now you can complete the remote control infos. You can use the EXPORT SHEET button.
    • You must edit the 'sheet' file using a text editor (e.g. Notepad++).
      Add all buttons ( NAME, ROW, COL and MODE). See as example the controls in demo remotesDB.
    • Keep attention to key names, use (where possible) only names in irp_actions.
    • If you do not define "modes" on remote control record, use the default value 'A' for all keys.
    • Then do "IMPORT SHEET" to import the data from SHEET back into remotesDB.
    • You can redo EXPORT/edit/IMPORT cycle until you are satisfied with the result. Because new sheets are without stream informations, do not use the EXECUTE button. Do not change name or dir to 'sheet' files.
      You can use the SHEET files as backup. In this case, better if you zip the files, to not overwrite it.

  6. After defined the remote control layout, you can capture 5 to 10 IR streams, using RAW CAPTURE tool.

  7. Verify the captured streams, using the EXECUTE button.

  8. Finally you can use DISCOVERY PROTOCOL to analyse the captured streams. You must choose 2 RAW from the new protocol and the tool will try to decode the RAWs using only compatible known protocols. Repeat the test many times with different RAWs.

  9. If you lucky and if you are sure about the new protocol identification, you can change it, from unknownXX to true name.
    As example, 'unknow01' (in original demo) is really 'NEC1_16', because: 1) uses dittos (that excludes NEC2, Pioneer) 2) uses D and ~D (and not D, S like NEC1).

  10. Last, you use verify EXECUTE, to verify and link all keys, and EXPORT SHEET as backup.
