Discovery protocols for remotesDB

This extension of remotesDB is for the management of new remote controls. It adds to remotesDB:
  • 100+ protocols with IRP, in table irp_protocols, and a new table irp_dataprotocols with statistical data.
  • A complete device type list to irp_devtypes.
  • A tool to test any of 100+ protocols with any data: HEX or dataset.
  • CRUD (Create, Read, Update, Delete) library, that allows fast and easy creation of new management pages for remotesDB tables.
  • A complete workflow for add new remote controls to remotesDB
Installation:
Precondition: last version (ver. 1.3, from http:/... ) of phpIRPlib and remoteDB installed and working.
  • Copy all files on www/remoteDBdiscovery and updates on phpIRPlib and remoteDB.
  • Import full_protocols.sql: It will add 100+ protocols to remotesDB.
  • Import irp_protocoldata.sql: It will add a new table to remotesDB.
  • Import view_protocoldata.sql: It will add a new view to remotesDB.

New management tools:

Using the simple Test protocols. you can test the 100+ protocols now in remotesDB using any data and get RAW stream.

The tool Verify/update irp_protocoldata automatizes the management of irp_protocoldata table. Use it in case of modified or new IRP

Workflow for new controls helps you to add a new IR remote control to remotesDB.
Test protocols.
You can insert data as HEX ('3A3B00F') or as data-set ('{D=35, F=0x2A}') and repetition number [1..4].
This tool encodes/decodes the RAW stream giving you some statistics.

Verify/update irp_protocoldata
For new protocols or in case of IRP changes.
This tool don't create new records, only updates existing records in irp_protocoldata table.
For a new protocol, you must create a new record on irp_protocoldata table, and set the fields: idprotocol, base_time, coding, before the use of this tool.
